# TELL — Psychological Warfare Chess

> Your heartbeat is visible. Your body betrays you. Bluffing is a skill.

Chess where both players' webcams run throughout the game, extracting heart rate via rPPG. Your opponent sees your panic meter live. You see theirs.

---

## Quick Start

```bash
pnpm install
pnpm setup:stockfish       # copies Stockfish WASM → public/stockfish/
cp .env.local.example .env.local
# fill in Firebase credentials (see below)
pnpm dev
```

## Firebase Setup

1. Create a Firebase project at https://console.firebase.google.com
2. Enable **Authentication → Google sign-in**
3. Enable **Firestore Database** (production mode, then set rules — see below)
4. Enable **Realtime Database** (production mode)
5. Add a web app, copy the config into `.env.local`

### Firestore Rules

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid} {
      allow read: if true;
      allow write: if request.auth.uid == uid;
    }
    match /usernames/{username} {
      allow read: if true;
      allow create: if request.auth != null;
    }
    match /games/{gameId} {
      allow read: if true;
      allow create, update: if request.auth != null;
    }
    match /friendRequests/{reqId} {
      allow read, create: if request.auth != null;
      allow update: if request.auth.uid == resource.data.to;
    }
    match /challenges/{cid} {
      allow read, create: if request.auth != null;
      allow update: if request.auth.uid == resource.data.to;
    }
  }
}
```

## rPPG Pipeline

The heart rate extraction runs entirely in the browser:

1. **MediaPipe FaceLandmarker** detects the face and gives landmark positions each frame
2. A 30×30 pixel ROI is extracted from the forehead (landmark #10)
3. Per-frame mean R, G, B values are sent to a **Web Worker** (`public/workers/rppg-worker.js`)
4. The worker maintains a 10-second rolling buffer at ~30fps
5. **CHROM algorithm** (de Haan & Jeanne 2013) extracts the blood volume pulse signal
6. A **2nd-order Butterworth bandpass filter** (0.7–4 Hz, coefficients precomputed via bilinear transform) isolates the cardiac band
7. An **FFT** on the filtered, Hann-windowed signal finds the dominant frequency
8. `BPM = peak_bin × (fps / fft_length) × 60`
9. An **EMA (α=0.15)** smooths the output
10. `panic_index = clamp((smoothed_bpm - baseline) / 20, 0, 1)` — only the float is sent to Firebase

The rPPG logic is entirely in `src/lib/rppg/` (TypeScript, for reference/testing) and `public/workers/rppg-worker.js` (the runtime).

## The "Tell" System

- **Panic index** = normalized delta above resting HR. 20+ BPM above baseline = 100% panic.
- **Raw BPM is never shown** — only the 0–1 relative index
- **Bluff token**: one per game. Freezes your displayed panic for 20s. Visible only to you. Strategic use: activate before a sacrifice so your opponent misreads your confidence.
- **Post-game reveal**: after the game, both players see each other's full HR timeline, annotated with the moves that caused the biggest spikes. Screenshot gold.
